API tokens

Use Bearer tokens to read, delete, and export feedback from scripts or other services. Tokens are shown once when created.

Copy this token now. It won’t be shown again.

API docs

Base URL is this server. Authenticate with a session cookie (dashboard) or Authorization: Bearer <token>.

POST /api/feedback

Public. Submit widget feedback. No auth.

curl -X POST $BASE/api/feedback \
  -H 'Content-Type: application/json' \
  -d '{"type":"comment","message":"Hello","pageUrl":"https://example.com"}'

GET /api/feedback

List feedback. Optional ?type=bug.

curl $BASE/api/feedback \
  -H "Authorization: Bearer $TOKEN"

DELETE /api/feedback/:id

Delete one item.

curl -X DELETE $BASE/api/feedback/ITEM_ID \
  -H "Authorization: Bearer $TOKEN"

GET /api/export.csv

Download all feedback as CSV.

curl $BASE/api/export.csv \
  -H "Authorization: Bearer $TOKEN" -o feedback.csv

GET /api/health

Public health check.
